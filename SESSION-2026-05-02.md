### 3. Post Screens — ACTIVE FOCUS

#### Offline Queue / Pending Rounds — DESIGN AGREED
- On submit: attempt POST → if fails (or no URL), auto-save round to localStorage as pending
- localStorage key: `pendingRounds[]` — array of round objects (18 holes + stats + date + course)
- Each round ~1–2KB; cap at 5 pending rounds max
- On app open: check for pending rounds in localStorage
  - If any exist: show NON-BLOCKING banner on Home screen
  - Banner is informational only — player can dismiss and start a new round regardless
  - Player may be on 1st tee with no service — must never block starting a new round
  - Multiple rounds can stack in the queue (up to cap of 5)
- If cap (5) reached: "Round not saved — 5 rounds are waiting to post. Post or discard one first."
- Pending rounds banner: "You have X round(s) waiting to post." + [Post Now] [Later] buttons
- [Post Now] → loads that round's data → goes to Post Screen pre-seeded, attempts repost
- [Later] → dismisses banner, user proceeds normally

#### Notification / Reminder strategy
- App-open banner covers 95% of use cases — no backend needed
- Web push notifications: possible via SW but requires user permission + push server — defer
- Email deep-link: needs user email + backend to send — future module, design TBD
  - URL scheme idea: `?action=post&round=abc` → PWA reads param on launch → routes to pending round
