## Lock-file mystery (recurring)

Sandbox commits leave `.git/index.lock`, `.git/HEAD.lock`, `.git/objects/maintenance.lock` behind that the sandbox itself cannot remove (`Operation not permitted` on unlink — likely a macOS extended-ACL or quarantine attribute issue).

Workaround: **don't commit from the sandbox**. Edit files via Python or the Edit tool, then commit + push from GitHub Desktop only. If lock still appears after that, run the rm with GitHub Desktop fully quit.

The `+` after the rwx bits on `index.html` and `START-HERE.md` from `ls -la` (extended ACLs) seems to be involved — worth investigating in a quiet moment with `xattr -l` and `chmod -N` if Paul wants this gone for good.
