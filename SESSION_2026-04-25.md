# SESSION 2026-04-25 — iPhone PUTTS bug fixed; layout refactored to elastic-stage

> Closes the iPhone-only rendering bug from earlier in the day and refactors the
> layout architecture to be robust against viewport variation.

---

## TL;DR

PUTTS row was missing on iPhone Safari because the `.screen` had `overflow: hidden`
and the footer's intrinsic height (with a 230px padding-bottom hack) exceeded the
remaining viewport space — so PUTTS was rendered into clipped territory below the
visible area. Fixed across v9.12 → v9.15. Layout is now: masthead fixed, white
stage **elastic**, footer **content-sized** with min-height matching across all
five screens for visual parity. Title font on non-hole screens now matches the
hole number size (56/46px). "Final Round Score" renamed to "Final Score" so it
fits at 46px on small screens.

## Diagnostic learnings (for future Claude / Paul)

- iOS Safari's `100dvh` doesn't reliably exclude the bottom URL bar overlay in
  every state — using elastic stage architecture sidesteps the entire question.
- `?v=N` cache-busts work for HTTP cache but not for SW cache hits; the only
  reliable cure for sticky SWs is to fix the SW strategy (network-first for HTML).
- "Headroom didn't change between padding bumps" was the smoking gun — meant the
  footer was being clipped by `overflow: hidden`, not just under-padded.
- Paul's iPhone is 15 Pro Max — viewport ~700-750px in Safari. Small-screen rule
  (`max-height: 750px`) applies.
- Cross-device responsive design (`env(safe-area-*)`, `100dvh`, elastic flex)
  works equally well on Android Chrome / Samsung Internet.
