### Diagnosis pointer
Since Mac Chrome (no SW) renders the new file correctly with all 6 rows, but iPhone Safari shows X-UD as last and never shifts: **the iPhone is almost certainly serving a stale cached version, despite the `?v=N` cache-bust URLs**. The previous SW (golf-scores-v9 or v10) may still be intercepting fetches.

### Next session — try in this order

1. **iPhone Settings → Apps → Safari → Advanced → Website Data → search "golf" → swipe-delete.** Then reload `?v=300`. This is the strongest possible cache wipe — also kills any registered service worker for that origin.
2. If still broken, ask Paul for an iPhone screenshot showing the bottom of the page with chrome menu open. Compare to Mac Chrome rendering. That will conclusively show whether it's a cache/SW issue or a real rendering bug.
3. If the screenshot shows the page DOES match Mac Chrome (PUTTS visible) — then the cache wipe fixed it.
